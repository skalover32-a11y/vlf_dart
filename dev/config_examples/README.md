# Архивные примеры конфигураций VLF VPN

Эта папка содержит **архивные и тестовые конфигурации** Clash Meta, сохранённые для справки и отладки.

⚠️ **НЕ ИСПОЛЬЗУЮТСЯ В RUNTIME** — приложение генерирует `config.yaml` динамически через `lib/clash_config.dart`.

## Назначение файлов

### Тестовые выходы
- `config_test*.yaml` — результаты выполнения `dart run tools/gen_config.dart` и `test_rules_priority.dart`
- `config_test_clash.yaml` — выход теста `tools/test_clash_config.dart`

### Примеры режимов
- `config_global*.yaml` — конфигурации в глобальном режиме (весь трафик → VPN)
  - `config_global_no_excl.yaml` — без исключений
  - `config_global_with_excl.yaml` — с исключениями по доменам/приложениям
- `config_rf*.yaml` — конфигурации в РФ-режиме (.ru/.su/.рф → DIRECT, остальное → VPN)
  - `config_rf_no_excl.yaml` — без исключений
  - `config_rf_with_excl.yaml` — с исключениями

### Исторические форматы
- `config_ru.yaml` — старый формат РФ-режима
- `config_test.json`, `config_no_reality.json` — старые JSON-форматы (sing-box)
- `example_config.yaml` — эталонный пример конфигурации

## Использование

Эти файлы можно использовать для:
1. **Сравнения** — проверить, как изменилась генерация конфигов после правок
2. **Отладки** — понять, какие правила применялись в конкретном сценарии
3. **Документации** — примеры корректной структуры Clash Meta config

Для **реального запуска** используйте:
```bash
dart run test_rules_priority.dart  # запустит все тесты и покажет актуальные конфиги
```

Или просто запустите приложение — `config.yaml` будет сгенерирован автоматически в `baseDir`.
